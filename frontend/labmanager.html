sudo cp spacecaptain-updater.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now spacecaptain-updater

cd ~/spacecaptain
BUILD_NR=$(git rev-list --count HEAD) docker compose up -d --build backend

SpaceCaptain
SpaceCaptain ist das Verwaltungssystem für deinen Makerspace — Berechtigungen, Maschinensteuerung per Smart Plug, Plug-Pool-Verwaltung und Maschinenpflege in einer App.
SICHERHEITSHINWEIS

SpaceCaptain ist für den Betrieb in einem internen Netzwerk (LAN) konzipiert. Eine direkte Erreichbarkeit aus dem Internet wird nicht empfohlen. Für externen Zugriff sollte ein VPN (z.B. WireGuard) vorgeschaltet werden.

🚀Erste Schritte
Anmelden als Lab Manager
Öffne SpaceCaptain im Browser. Auf der Login-Seite Benutzername (E-Mail) und Passwort eingeben. Nach erfolgreicher Anmeldung wird das Dashboard angezeigt. Die Session bleibt für die konfigurierte JWT-Dauer aktiv (Standard: 8 Stunden).
Tipp: Der Browser merkt sich die Session. Nach einem Neustart des Tabs kann eine erneute Anmeldung nötig sein.
Dashboard verstehen
Das Dashboard zeigt auf einen Blick: aktive Maschinen-Sessions, ausstehende Gast-Registrierungen, aktive Wartelisten-Einträge und fällige Wartungen. Einträge mit einem !-Badge in der Sidebar signalisieren dringenden Handlungsbedarf.
Passwort ändern
Eigenes Passwort ändern: Sidebar → Lab Manager → eigenen Eintrag bearbeiten (✏) → neues Passwort eingeben → Speichern. Fremde Passwörter können nur Admins ändern.
Rollen: Admin vs. Manager
  • Admin — hat Zugriff auf alle Funktionen inkl. Einstellungen und kann andere Lab Manager verwalten.
  • Manager — kann Gäste, Maschinen und Berechtigungen verwalten, hat aber keinen Zugriff auf Einstellungen oder die Admin-Funktionen.
Seiten & URLs
SpaceCaptain stellt mehrere Seiten bereit, die unterschiedliche Zielgruppen ansprechen:
URL Zweck
/ Gäste-Login und Maschinenzugang (QR-Code-Ziel)
/labmanager Admin- und Manager-Interface (diese Seite)
/display Anzeigetafel — Maschinenübersicht für TV/Kiosk
/register Gäste-Registrierungsformular (per Link oder QR-Code teilbar)
/nfc-setup NFC-Writer Einrichtung (ESP32-Flasher)
/docs REST-API Dokumentation (OpenAPI / Swagger)
👥Gästeverwaltung
Gast manuell anlegen
  1. Sidebar → Gäste → «+ Gast hinzufügen»
  2. Vor- und Nachname, Benutzername und Passwort eingeben
  3. Optional: E-Mail und Telefon
  4. «Speichern» — der Gast ist sofort aktiv
Tipp: Der Benutzername muss einzigartig sein und wird zum Login verwendet. Empfehlung: vorname.nachname
Selbstregistrierung freischalten oder ablehnen
Wenn ein Gast sich über den QR-Code auf der Display-Seite selbst registriert, erscheint er mit einem !-Badge im Dashboard unter «Ausstehende Registrierungen».
  1. Dashboard → Abschnitt «Ausstehende Registrierungen»
  2. Freischalten: Gast erhält sofort Zugang (kein Maschinenrecht, muss separat vergeben werden)
  3. Ablehnen: Datensatz wird gelöscht
Gast sperren vs. deaktivieren
  • Sperren (rotes Schloss-Icon): Der Gast kann sich einloggen, aber keine Maschinen mehr benutzen. Berechtigungen bleiben erhalten.
  • Deaktivieren (Schalter aus): Der Gast kann sich nicht mehr einloggen.
Hinweis: Das Sperren eines Gastkontos beendet keine laufende Session. Die Maschine läuft bis zum manuellen oder automatischen Ende weiter.
Gast-Token (NFC/QR-Login)
Gäste können sich statt mit Passwort auch per NFC-Token oder QR-Code authentifizieren. Tokens werden automatisch vergeben und haben eine konfigurierbare Gültigkeitsdauer (Standard: 30 Tage). Nach Ablauf muss sich der Gast erneut mit Passwort anmelden.
Warteliste
Gäste können sich für eine Maschine in die Warteliste eintragen, wenn diese gerade belegt ist.
  • Sobald die Maschine frei wird, rückt der erste Gast vor und erhält eine ntfy-Push-Benachrichtigung (sofern eingerichtet)
  • Das Reservierungsfenster ist konfigurierbar (Einstellungen → «Warteschlangen-Reservierungsfenster»). Meldet sich der Gast nicht innerhalb dieser Zeit an der Maschine an, verfällt sein Platz und der Nächste wird benachrichtigt
  • Lab Manager können neue Einträge manuell über das Dashboard anlegen
Voraussetzung für ntfy: Der Gast muss sein persönliches ntfy-Topic in der ntfy-App abonniert haben (Gäste-Seite → Scan-Screen).
Login-Link für Gäste generieren
Statt eines Passworts kann ein direkter Login-Link erstellt und dem Gast zugesendet werden — nützlich bei vergessenen Zugangsdaten oder für Gäste ohne Passwort.
  1. Sidebar → Gäste → Gast bearbeiten (✏)
  2. Abschnitt «Login-Token» → «Neuer Token»
  3. Generierten Link kopieren und dem Gast per E-Mail oder Messenger schicken
Ein Klick auf den Link loggt den Gast direkt ein — ohne Passworteingabe. Der Link bleibt für die konfigurierte Token-Gültigkeitsdauer aktiv (Standard: 365 Tage).
Hinweis: Den Link sicher übermitteln — wer den Link hat, kann sich als dieser Gast einloggen.
🏭Maschinen
Maschine anlegen
  1. Sidebar → Maschinen → «+ Maschine hinzufügen»
  2. Name, Kategorie, Hersteller und Modell eingeben
  3. Optional: Standort, Kommentar, Smart Plug konfigurieren
  4. «Schulung erforderlich» aktivieren wenn Gäste eine explizite Berechtigung brauchen
Tipp: Maschinen ohne «Schulung erforderlich» sind für alle aktiven Gäste automatisch zugänglich — sofern sie nicht explizit gesperrt wurden.
Smart Plug zuweisen
Smart Plugs werden zentral im Plug-Pool verwaltet und dann Maschinen zugewiesen. Eine Maschine kann mehrere Plugs haben (z.B. CNC mit zwei Steckdosen).
  1. Sidebar → Ausstattung → Plug-Pool → Plug anlegen (falls noch nicht vorhanden)
  2. Sidebar → Maschinen → Maschine bearbeiten (✏)
  3. Abschnitt «Smart Plugs» → Dropdown «Plug hinzufügen» öffnen und Plug auswählen → «+ Hinzufügen»
  4. Für weitere Plugs: Schritt 3 wiederholen
  5. Optional: Leerlauf-Schwellwert, Timeout und Poll-Intervall konfigurieren
  6. Speichern
Der mit ★ markierte Plug ist der Primär-Plug — er wird für die Leerlauf-Erkennung und den Status-Poll verwendet. Alle weiteren Plugs schalten gleichzeitig mit.
Hinweis: Der Smart Plug muss im gleichen Netzwerk wie der SpaceCaptain-Server erreichbar sein. Feste IP-Adresse empfohlen (DHCP-Reservierung im Router).
Mehrere Plugs pro Maschine (z.B. CNC)
Maschinen, die mehr als eine Steckdose benötigen (z.B. CNC-Fräse > 2000 W), können mehrere Plugs zugewiesen bekommen. Beim Ein- und Ausschalten werden alle Plugs der Maschine gleichzeitig geschaltet.
  • ★ Primär-Plug (erster zugewiesener Plug): Wird für Leerlauf-Erkennung, Status-Anzeige und Poll verwendet
  • Sekundär-Plugs: Schalten mit dem Primär-Plug, werden aber nicht separat überwacht
  • Die Reihenfolge wird durch die Zuweisung bestimmt — zuerst zugewiesener Plug ist immer Primär
Plug entfernen: Im Bearbeitungsdialog → «×» neben dem Plug klicken → Speichern. Wird der Primär-Plug entfernt, rückt der nächste Plug automatisch nach.
Leerlauf-Erkennung
Wenn Leerlauf-Schwellwert und Timeout konfiguriert sind, überwacht SpaceCaptain den Stromverbrauch. Fällt die Leistung unter den Schwellwert, startet ein Countdown. Nach Ablauf des Timeouts wird die Maschine automatisch abgeschaltet und die Session beendet. Die Leerlauf-Erkennung greift erst 60 Sekunden nach Session-Start, um Anlaufzeit zu ermöglichen.
Maschinenstatus manuell setzen
Mögliche Zustände: Online (verfügbar), Offline (nicht verwendbar), Wartung (in Reparatur/Service). Status ändern: Maschine bearbeiten → Feld «Status». Maschinen im Status «Offline» oder «Wartung» können von Gästen nicht gestartet werden.
QR-Code und NFC-Tag beschreiben
Jede Maschine hat einen eindeutigen QR-Token. Den QR-Code drucken:
  1. Maschine in der Liste → QR-Icon
  2. QR-Code erscheint als Bild — rechtsklick → Bild speichern oder drucken
NFC-Tag beschreiben:
  1. NFC-Writer mit SpaceCaptain verbunden und bereit (LED blinkt blau)
  2. Maschine bearbeiten → «NFC beschreiben»
  3. NFC-Tag ans Gerät halten — LED leuchtet grün bei Erfolg
Session manuell starten (Lab Manager)
Lab Manager können eine Maschinen-Session stellvertretend für einen Gast oder auf eigene Rechnung starten — z.B. um eine Maschine einzurichten oder kurz zu testen.
  1. Dashboard → Maschinen-Karte → «▶ Starten»
  2. Optional: Gast auswählen, dem die Session zugeordnet wird
  3. Bestätigen
Die Session erscheint im Aktivitätslog mit Vermerk des Lab Managers. Beenden über «■ Beenden» in der Dashboard-Karte.
Maschinenkategorien verwalten
Kategorien gruppieren Maschinen thematisch (z.B. Laser, CNC, 3D-Druck) und bestimmen die Anzeigereihenfolge auf der Display-Seite.
  1. Sidebar → Maschinen → «⚙ Kategorien»
  2. Neu: Name und Icon-Emoji eingeben → «Hinzufügen»
  3. Reihenfolge: per Pfeile anpassen
  4. Löschen: nur möglich wenn keine Maschine diese Kategorie noch verwendet
Maschinenliste exportieren / importieren (CSV)
Export: Sidebar → Maschinen → «↓ CSV». Die Datei enthält Name, Kategorie, Hersteller, Modell, Seriennummer, Standort, Status, Schulung und Kommentar. Kann in Excel oder LibreOffice Calc geöffnet werden.

Import: Sidebar → Maschinen → «↑ CSV». Eine CSV-Datei im gleichen Format (Semikolon-getrennt, Spaltenüberschriften in der ersten Zeile) auswählen. Es erscheint eine Vorschau mit allen Zeilen und ob sie importiert oder übersprungen werden (bereits vorhandene Namen werden ignoriert). Erst nach Bestätigung werden die Daten gespeichert.

Hinweis: Plug-Konfiguration und ID-Spalte werden beim Import ignoriert. Plugs können nach dem Import in den Maschineneinstellungen zugewiesen werden.
Sicherheitshinweise hinterlegen
Für Maschinen mit besonderen Schutzanforderungen (z.B. Tischkreissäge, Laser) können Sicherheitshinweise hinterlegt werden. Der Gast muss diese vor dem Einschalten aktiv bestätigen.
  1. Maschine bearbeiten (✏) → Feld «Sicherheitshinweise» ausfüllen
  2. Hinweise eingeben — Freitext, z.B.:
    Schutzbrille · Gehörschutz · Haare zurückbinden
  3. Speichern
Beim nächsten Einschalten erscheint der Text in einem orangefarbenen Hinweis-Panel. Der Gast muss «Ich habe die Sicherheitshinweise gelesen und befolge sie» anhaken bevor der Einschaltenknopf aktiv wird.
Tipp: Feld leer lassen = kein Bestätigungs-Dialog. Der normale Einschaltfluss wird nur bei befülltem Feld unterbrochen.
🔌Plug-Pool
Was ist der Plug-Pool?
Der Plug-Pool ist eine zentrale Verwaltung für alle Smart Plugs im Makerspace. Jeder Plug wird einmalig mit Name, Typ und IP-Adresse erfasst und kann danach beliebigen Maschinen zugewiesen werden. So müssen Verbindungsdaten nicht mehrfach gepflegt werden und der Überblick bleibt gewahrt — auch bei vielen Plugs.
  • Unterstützte Typen: myStrom, Shelly Gen1, Shelly Gen2/Gen3/Gen4
  • Ein Plug kann mehreren Maschinen zugewiesen sein (alle schalten dann gleichzeitig)
  • Eine Maschine kann mehrere Plugs haben (alle schalten gleichzeitig)
  • Freie Plugs (ohne Zuweisung) können direkt aus dem Pool testweise geschaltet werden
Plug hinzufügen
  1. Sidebar → Ausstattung → Plug-Pool → «+ Plug hinzufügen»
  2. Name eingeben (z.B. «Shelly Laser» oder «myStrom CNC»)
  3. Typ wählen: myStrom, Shelly Gen1 oder Shelly Gen2+
  4. IP-Adresse des Plugs eingeben
  5. Optional: Auth-Token (myStrom: API-Token; Shelly: admin:passwort)
  6. Optional: Notiz für interne Zwecke
Tipp: Feste IP-Adressen im Router reservieren (DHCP-Reservierung), damit die Adressen nach einem Neustart gleich bleiben.
Auth-Token konfigurieren
  • myStrom: In der myStrom-App unter Einstellungen → API-Token einsehen. Token ins Feld «Auth-Token» eintragen.
  • Shelly Gen1: Wenn ein HTTP-Passwort in der Shelly-Web-UI gesetzt ist, Token im Format admin:passwort eintragen.
  • Shelly Gen2/Gen3/Gen4: Wenn ein Passwort gesetzt ist, Token im Format admin:passwort eintragen. Ohne Passwortschutz kann das Feld leer bleiben.
Hinweis: Der Token wird in der Plug-Pool-Übersicht nie im Klartext angezeigt — nur ob einer hinterlegt ist (••••••) oder nicht (—).
Plug einer Maschine zuweisen
Die Zuweisung erfolgt in der Maschinenkonfiguration — nicht im Plug-Pool selbst.
  1. Sidebar → Maschinen → Maschine bearbeiten (✏)
  2. Abschnitt «Smart Plugs» → Dropdown «Plug hinzufügen» → Plug auswählen → «+ Hinzufügen»
  3. Für weitere Plugs: Schritt 2 wiederholen
  4. Speichern
Nach der Zuweisung erscheint der Maschinenname in der Plug-Pool-Tabelle unter «Zugewiesen an». Der zuerst zugewiesene Plug ist automatisch der Primär-Plug (★).
Shared Plug: Ist ein Plug bereits einer anderen Maschine zugewiesen, erscheint eine Bestätigung. Beide Maschinen schalten dann gleichzeitig — sinnvoll z.B. für einen ganzen Arbeitsplatz.
Plug-Zuweisung aufheben
  1. Sidebar → Maschinen → Maschine bearbeiten (✏)
  2. Abschnitt «Smart Plugs» → «×» neben dem Plug klicken
  3. Speichern
Wird der Primär-Plug (★) entfernt und es gibt weitere Plugs, rückt der nächste automatisch als Primär nach. Sind alle Plugs entfernt, erscheint der Plug in der Pool-Liste wieder als «— frei —».
Plug testweise schalten (Identifikation)
Um physisch festzustellen, welcher Plug im Regal welcher Steckdose entspricht, können freie Plugs direkt aus dem Pool ein- und ausgeschaltet werden.
  1. Sidebar → Ausstattung → Plug-Pool
  2. In der Zeile eines freien Plugs (ohne Zuweisung): «▶ EIN» oder «■ AUS» klicken
Nur für freie Plugs: Bereits zugewiesene Plugs können nur über die Maschinen-Session oder das Dashboard geschaltet werden. Das verhindert unbeabsichtigte Unterbrechungen laufender Sessions.
Plug löschen
Ein Plug kann nur gelöscht werden, wenn er keiner Maschine zugewiesen ist. Zuerst die Zuweisung in der Maschinenkonfiguration aufheben, dann im Plug-Pool auf 🗑 klicken.
Filtern und sortieren
Die Plug-Pool-Liste kann gefiltert werden nach:
  • Freitext: Name, IP-Adresse oder Notiz
  • Typ: myStrom / Shelly Gen1 / Shelly Gen2+
  • Zuweisung: nur freie oder nur zugewiesene Plugs
Ein Klick auf einen Spaltenkopf (Name, Typ, IP-Adresse, Zugewiesen an) sortiert die Liste auf- oder absteigend.
🔑Berechtigungen
Berechtigung vergeben
  1. Sidebar → Berechtigungen
  2. Maschine auswählen → Gast aus der Liste wählen → «Zugang gewähren»
Alternativ direkt über den Gast-Eintrag: Gäste → Gast bearbeiten → Berechtigungen-Tab.
Tipp: Bei Maschinen ohne Schulungspflicht brauchen Gäste keine explizite Berechtigung — der Zugang ist automatisch gewährt, sofern kein aktives Verbot besteht.
Berechtigung entziehen vs. sperren
  • Entzug: Die Berechtigung wird vollständig entfernt. Bei Maschinen ohne Schulungspflicht hat der Gast danach wieder automatischen Zugang.
  • Sperren: Setzt ein explizites Verbot. Gilt auch bei Maschinen ohne Schulungspflicht. Geeignet wenn ein Gast eine bestimmte Maschine definitiv nicht nutzen darf.
Berechtigungen für mehrere Gäste gleichzeitig setzen
In der Maschinenansicht können Berechtigungen für mehrere Gäste auf einmal vergeben oder entzogen werden:
  1. Sidebar → Berechtigungen → Ansicht «nach Maschine» wählen
  2. Maschine auswählen
  3. Mehrere Gäste per Checkbox markieren
  4. «Zugang gewähren» oder «Zugang entziehen» für alle Markierten anwenden
Tipp: Ideal nach einer Schulung — alle Teilnehmenden auf einmal freischalten.
⚡Automationen
Was sind Automationen?
Automationen führen eine Aktion automatisch aus — basierend auf kombinierten Bedingungen. Alle Bedingungen einer Regel müssen gleichzeitig wahr sein (AND-Verknüpfung).
  • Mehrere Regeln können unabhängig voneinander feuern
  • Optional: Nachlaufzeit — wie lange nach Bedingungsende noch gewartet wird (nur bei Maschinen-Aktionen relevant)
Aktionstypen
  • 🔌 Maschine schalten: Schaltet eine Ziel-Maschine (mit Smart Plug) ein, wenn alle Bedingungen erfüllt sind — und aus, wenn sie nicht mehr erfüllt sind. Mit Nachlaufzeit möglich.
  • 🟢 Raum öffnen: Öffnet den Raum einmalig, sobald alle Bedingungen erstmals erfüllt werden. Ideal für zeitgesteuertes Öffnen am Morgen.
  • 🔴 Raum schliessen: Schliesst den Raum einmalig, sobald alle Bedingungen erstmals erfüllt werden. Maschinen mit «Bei Raumschluss ausschalten» werden abgeschaltet, laufende Sessions bleiben aktiv.
  • 🔔 Push-Nachricht: Sendet eine ntfy-Push-Nachricht an ein konfiguriertes Manager-Topic, sobald alle Bedingungen erstmals erfüllt werden.
Zeitgesteuerter Raum: Für automatisches Öffnen und Schliessen je eine separate Regel mit «Raum öffnen» resp. «Raum schliessen» erstellen — z.B. Zeitplan Mo–Fr 08:00–18:00 für Öffnen, 18:00–23:59 für Schliessen.
Regel erstellen
  1. Sidebar → Automationen → «+ Neue Regel»
  2. Name vergeben (optional), Aktionstyp wählen
  3. Je nach Aktionstyp: Ziel-Maschine, ntfy-Topic und Nachrichtentext wählen
  4. Über «+ Bedingung» eine oder mehrere Bedingungen hinzufügen
  5. Nachlaufzeit einstellen (nur für Maschinen-Aktionen, 0 = sofort ausschalten)
Bedingungstypen
  • ⚡ Leistung: Eine Quell-Maschine überschreitet einen Watt-Schwellwert. Einschalten ab X W, Ausschalten unter Y W (Hysterese verhindert Flackern).
    Beispiel: Tischkreissäge ≥ 150 W → Absaugung EIN. Nachlauf 60 s damit Späne vollständig abgesaugt werden.
  • 🕐 Zeitplan: Aktuelle Zeit liegt im konfigurierten Fenster (Wochentage + Von–Bis).
    Beispiel: Mo–Fr 08:00–18:00 → Kaffeemaschine EIN.
  • 🏠 Raum geöffnet: Der Raum-Öffnungsstatus im Dashboard ist aktiv. Kann manuell oder automatisch beim ersten Gast-Login des Tages gesetzt werden.
  • 👤 Aktive Session: Mindestens eine Maschinen-Session ist aktiv. Nützlich um Geräte nur einzuschalten solange jemand im Space arbeitet.
    Mit Nachlaufzeit kombinierbar: Session-aktiv + Nachlauf 30 min = Lüftung läuft 30 min nach letzter Session weiter.
Raum-Öffnungsstatus
Der Raum-Status ist oben im Dashboard sichtbar. Er kann:
  • Manuell über «🟢 Öffnen» / «🔴 Schliessen» gesetzt werden
  • Automatisch beim ersten Gast-Login des Tages geöffnet werden (konfigurierbar unter Einstellungen → «Automatisch öffnen»)
Regeln mit der Bedingung «Raum geöffnet» werden nur ausgeführt wenn der Status aktiv ist — ideal für Spaces die nicht täglich geöffnet sind.
Watcher-Status (Live-Anzeige)
In der Regelliste wird der aktuelle Watcher-Status angezeigt:
  • ● AN — alle Bedingungen erfüllt und Aktion ausgelöst (bei Maschine: Plug EIN; bei Raum/Push: einmalig gefeuert)
  • ⏱ Xs — Bedingungen nicht mehr erfüllt, Nachlauf läuft (nur Maschinen-Aktionen)
  • ● AUS — Bedingungen nicht erfüllt, Regel wartet
Raum- und Push-Aktionen feuern einmalig beim Übergang in den AN-Zustand und setzen sich automatisch zurück, sobald die Bedingungen nicht mehr erfüllt sind — damit sie am nächsten Tag erneut feuern können.
Regel deaktivieren / löschen
  • Deaktivieren: Checkbox «Aktiv» — Regel bleibt gespeichert, wird nicht ausgeführt
  • Löschen: ✕-Symbol — dauerhaft entfernt
Beim Deaktivieren oder Löschen wird die Ziel-Maschine nicht automatisch ausgeschaltet.
🔧Maschinenpflege
Wartungsintervall anlegen
  1. Sidebar → Maschinenpflege → «+ Neues Intervall»
  2. Maschine wählen, Bezeichnung eingeben (z.B. «Sägeblatt prüfen»)
  3. Intervall definieren: nach Betriebsstunden und/oder nach Kalendertagen
  4. Optional: Vorwarnzeit — ab wann soll die Warnung erscheinen
Wartung durchführen und dokumentieren
  1. Maschinenpflege → fälliges Intervall auswählen → «✓ Durchgeführt»
  2. Datum, ausführende Person und optionale Notizen eingeben
  3. «Speichern» — der Zähler wird zurückgesetzt
Für Wartungen ohne Intervall (spontan): «+ Freie Wartung» im Abschnitt Wartungsintervalle.
Statusfarben verstehen

  • Orange / Warnung: Wartung steht bald an (innerhalb Vorwarnzeit)
  • Kein Badge: Alles im grünen Bereich
Fällige Wartungen erscheinen auch auf dem Dashboard mit einem !-Badge in der Sidebar.
📢Aushänge
Aushang erstellen
Sidebar → Aushänge → «+ Neuer Aushang».
  • Einmalig: gilt von einem Start- bis Endzeitpunkt
  • Wiederkehrend: gilt an bestimmten Wochentagen zu festgelegten Zeiten — ideal für regelmässige Öffnungszeiten oder Kurshinweise
  • Anzeigetyp Banner: erscheint als farbiger Balken auf dem Display und der Gästeseite
Ankündigung (Display-Overlay)
Zusätzlich zu den terminierten Aushänge gibt es in Einstellungen → Display ein statisches Ankündigungs-Textfeld und einen Lauftext-Ticker. Diese erscheinen dauerhaft auf der Display-Seite bis sie gelöscht werden.
🔔Push-Nachrichten
Wie Push-Nachrichten funktionieren
SpaceCaptain nutzt ntfy für Push-Nachrichten. Jeder Gast erhält automatisch ein eigenes ntfy-Topic. Push-Nachrichten werden ausgelöst, wenn:
  • Eine Maschine in der Warteschlange frei wird («Du bist dran!»)
  • Ein Notfall-Alarm ausgelöst wird
  • Eine Automationsregel mit Aktionstyp «Push-Nachricht» feuert
ntfy einrichten (Einstellungen)
Sidebar → Einstellungen → Push-Nachrichten:
  1. ntfy-Server: Standard ist https://ntfy.sh. Eigenen Server einsetzen für mehr Datenschutz.
  2. ntfy-Token: Nur nötig wenn der Server eine Authentifizierung verlangt
Tipp: Für produktiven Betrieb empfiehlt sich ein selbst gehosteter ntfy-Server, damit keine Gästedaten an externe Dienste übermittelt werden.
Gast mit ntfy verbinden
Jeder Gast hat ein eigenes Topic (sichtbar in der Gästedetail-Ansicht). Der Gast installiert die ntfy-App (iOS/Android) oder öffnet die Web-Oberfläche und abonniert sein Topic. Danach erhält er automatisch Push-Nachrichten.
Das Topic-URL wird dem Gast auch direkt in der Gäste-App unter «Einstellungen» als QR-Code angezeigt.
ntfy-Topics für Lab Manager konfigurieren
Neben den automatischen Gast-Topics können eigene ntfy-Kanäle für betriebliche Push-Nachrichten hinterlegt werden (z.B. für Notfall-Alarm-Meldungen an das Team).
  1. Sidebar → Push-Nachrichten → «+ Topic hinzufügen»
  2. Anzeigename und Topic-String eingeben (geheimen, schwer erratbaren String wählen)
  3. QR-Code scannen mit der ntfy-App um das Topic zu abonnieren
  4. Das Topic steht dann unter Einstellungen → Notfall-Alarm → ntfy-Topic und in Automationen → Aktionstyp «Push-Nachricht» zur Auswahl
Unterschied zu Gast-Topics: Gast-Topics sind automatisch generiert für persönliche Push-Nachrichten (Warteschlange). Manager-Topics sind manuell konfigurierte Kanäle für Alarm- und Automationsmeldungen.
🚨Notfall-Alarm
Notfall-Alarm konfigurieren
Sidebar → Einstellungen → Notfall-Alarm:
  • Auslöse-Token: Geheimes Token für den physischen Auslöse-Knopf (z.B. Shelly-Button)
  • Anzeigetext: Wird gross auf der Display-Seite angezeigt wenn der Alarm aktiv ist
  • ntfy-Nachricht: Wird an konfigurierte Lab-Manager-Topics gesendet
  • Dauer: Wie lange der Alarm automatisch aktiv bleibt (0 = manuell quittieren)
  • Smart Plug 1 & 2: Werden beim Alarm eingeschaltet — z.B. Sirene und Blinklicht
Alarm auslösen
Der Alarm wird über einen physischen Knopf ausgelöst (z.B. Shelly-Button), der den Auslöse-Endpunkt des Servers aufruft.
Wichtig: Beim Auslösen werden sofort die konfigurierten Smart Plugs (Sirene/Blinklicht) eingeschaltet, Lab Manager per ntfy benachrichtigt und der Alarm auf allen Display-Seiten angezeigt.
Shelly-Button konfigurieren
Der Shelly muss beim Tastendruck einen HTTP-POST an den SpaceCaptain-Server senden.
Endpunkt
POST https://<server-ip>/api/emergency/trigger
Header
X-Emergency-Token: <dein-token>
Shelly Gen2/Gen3/Gen4 (Plus, Pro, Mini):
  1. Shelly-Web-UI öffnen → Scripts → Neues Script erstellen
  2. Beim Tastendruck Shelly.call("HTTP.Post", ...) mit URL und Header aufrufen
  3. Alternativ: Actions → Button turned ON → URL eintragen (nur ohne Custom-Header)
Shelly Gen1:
  1. Shelly-Web-UI öffnen → Actions → Button turned ON
  2. URL eintragen: https://<server-ip>/api/emergency/trigger
  3. Custom-Header werden von Gen1 nicht unterstützt — Token muss als Query-Parameter übergeben werden (serverseitige Anpassung erforderlich)
Tipp: Für Gen2+ empfiehlt sich ein Script, da damit der Header X-Emergency-Token korrekt mitgesendet werden kann.
Alarm quittieren
Sidebar → Einstellungen → Notfall-Alarm → «Alarm quittieren». Es wird ein Kommentar und der Name des quittierenden Lab Managers ins Aktivitätslog eingetragen. Die Smart Plugs werden nicht automatisch wieder eingeschaltet — dies muss manuell erfolgen.

Maschinennutzung auswerten

⚙️Einstellungen
NFC-Writer URL
IP-Adresse des NFC-Writers im Format http://192.168.1.x. Die IP findet man im Serial Monitor nach dem ersten Start des ESP32, oder im Router unter «verbundene Geräte». Eine feste IP (DHCP-Reservierung) empfiehlt sich, damit die URL nicht wechselt.
JWT-Ablaufzeit
Bestimmt wie lange eine Lab-Manager-Session gültig ist (in Minuten). Standard: 480 (8 Stunden). Nach Ablauf wird man automatisch ausgeloggt. Für geteilte Geräte empfiehlt sich ein kürzerer Wert.
Achtung: Nach Änderung dieser Einstellung werden alle aktiven Sessions ungültig — alle eingeloggten Lab Manager müssen sich neu anmelden.
Warteschlangen-Reservierungsfenster
Zeit (in Minuten) die ein Gast hat, eine Maschine einzuschalten nachdem er benachrichtigt wurde. Startet er die Maschine nicht innerhalb dieser Zeit, wird sein Platz in der Warteschlange freigegeben.
Gast-Token-Gültigkeit
Bestimmt wie lange ein per E-Mail versendeter Login-Link (Token-URL) für Gäste gültig ist — in Tagen. Standard: 365 Tage. Nach Ablauf muss ein neuer Link generiert werden (Gästedetail → «Login-Link»).
Für sicherheitssensible Umgebungen empfiehlt sich ein kürzerer Wert, z.B. 30 Tage.
Dashboard-Aktualisierung
Intervall (in Sekunden) in dem das Lab-Manager-Dashboard automatisch neu lädt. Standard: 30 Sekunden. Bei sehr vielen Maschinen oder langsamen Verbindungen kann ein höherer Wert die Last reduzieren.
Modal-Verhalten (Backdrop)
Steuert ob ein Klick auf den abgedunkelten Hintergrund (Backdrop) ein offenes Fenster schliesst:
  • Eingabe-Modals: Formulare zum Anlegen/Bearbeiten — Standard: ein (Klick schliesst)
  • Anzeige-Modals: Detailansichten ohne Formular — Standard: ein
Deaktivieren verhindert versehentliches Schliessen bei ungespeicherten Eingaben.
Allgemeine Geschäftsbedingungen (AGB)
Unter Einstellungen → Allgemeine Geschäftsbedingungen kann ein AGB-Text im Markdown-Format hinterlegt werden. Ist ein Text vorhanden, erscheint auf der Registrierungsseite eine Pflicht-Checkbox — Gäste müssen die AGB vor der Registrierung akzeptieren.
  • Leer lassen = keine AGB-Pflicht
  • Markdown unterstützt: **fett**, *kursiv*, ## Überschriften, - Listen
Makerspace-Name
Der Name des Makerspace erscheint auf allen Seiten unter dem SpaceCaptain-Logo — auf der Gäste-Seite, dem Lab-Manager-Interface und der Display-Seite.
  1. Sidebar → Einstellungen → System
  2. Feld «Makerspace-Name» ausfüllen (z.B. «FabLab Musterstadt»)
  3. «Speichern»
Leer lassen zeigt keinen Untertitel unter dem Logo.
💾Backup & Restore
Automatisches Backup
SpaceCaptain erstellt täglich automatisch ein Backup — sofern aktiviert. Konfiguration unter Einstellungen → Automatisches Backup:
  • Aktivieren: Häkchen setzen
  • Uhrzeit: Zeitpunkt der täglichen Sicherung (z.B. 03:00)
  • Aufbewahrung: Anzahl Backups, die gespeichert bleiben (Standard: 30 ≈ 1 Monat)
Das Backup-Verzeichnis wird via BACKUP_DIR in der .env konfiguriert. Standard: ./backups. NFS-Shares werden unterstützt — einfach den gemounteten Pfad eintragen.
Tipp: Manuell kann jederzeit über Sidebar → Backup → «Jetzt sichern» ein Backup ausgelöst werden.
Backup herunterladen
Sidebar → Backup → in der Dateiliste auf ⬇ Download klicken. Die Datei wird als JSON heruntergeladen.
Sicherheit: Backup-Dateien enthalten Passwort-Hashes. Sicher aufbewahren und nicht per E-Mail versenden.
Backups löschen
Einzeln: Datei in der Liste → ✕.
Mehrere auf einmal: Checkboxen auswählen (oder «Alle» in der Kopfzeile), dann «X löschen» oben rechts klicken.
Restore aus gespeichertem Backup
Sidebar → Backup → in der Dateiliste auf ↩ Restore klicken.

Es gibt zwei Modi:
  • Additiv (Standard): Nur neue Einträge werden importiert — bestehende Daten bleiben unverändert. Ideal für Migrationen auf einen neuen Server.
  • Überschreiben: Bestehende Einträge werden mit den Werten aus dem Backup aktualisiert. Nützlich um einen früheren Zustand wiederherzustellen.
Tipp: Sessions und Aktivitätslog werden nie überschrieben — sie sind reine Verlaufsdaten.
Restore aus externer Datei (Import)
Sidebar → Backup → Abschnitt «Manueller Import» → JSON-Datei auswählen → «Importieren». Auch hier ist «Bestehende Einträge überschreiben» wählbar.
Was ist im Backup enthalten?
Enthalten: Einstellungen, Benutzer, Gäste, Maschinen, Plug-Pool, Berechtigungen, Sessions, Aktivitätslog, Wartungsintervalle, Wartungshistorie, Aushänge, ntfy-Topics, Automationsregeln.

Nicht enthalten: Aktive Warteschlangen-Einträge und NFC-Gast-Tokens (temporäre Laufzeitdaten, die nach einem Restore automatisch neu erstellt werden).
🛠Häufige Probleme
SpaceCaptain nicht erreichbar nach Server-Neustart
Der Backend-Container wartet beim Start bis die Datenbank bereit ist (bis zu 30 Sekunden, 10 Versuche). Wenn SpaceCaptain direkt nach einem Neustart nicht reagiert: 30 Sekunden warten und erneut versuchen. Hält das Problem an:
  1. SSH auf den Server
  2. docker compose logs backend prüfen
  3. Falls DB-Fehler: docker compose restart backend
Maschine schaltet nicht ein/aus
  1. Smart Plug erreichbar? Browser öffnen, Plug-IP direkt aufrufen (http://192.168.1.x)
  2. Plug im gleichen Netzwerk wie Server?
  3. IP-Adresse geändert? Im Plug-Pool (Ausstattung → Plug-Pool → ✏) aktualisieren — Änderung wird automatisch auf die zugewiesene Maschine übertragen
  4. Maschinen-Status auf «Online»?
Tipp: Im Aktivitätslog sieht man Fehler beim Schalten mit genauer Fehlermeldung.
Gast kann sich nicht einloggen
Mögliche Ursachen:
  • Konto deaktiviert → Gäste → Konto aktivieren
  • Falsches Passwort → Gäste → Passwort zurücksetzen
  • Ausstehende Registrierung → noch nicht freigeschaltet
  • NFC-Token abgelaufen → Gast muss sich einmalig mit Passwort anmelden um neuen Token zu erhalten
Display-Seite zeigt falschen Status
Die Display-Seite aktualisiert sich automatisch im konfigurierten Intervall (Standard: 30 Sekunden). Bei längeren Verzögerungen:
  1. Seite manuell neu laden
  2. Refresh-Intervall in den Einstellungen prüfen
  3. Browser-Konsole auf Fehler prüfen (F12)
NFC-Writer reagiert nicht
  1. LED-Status prüfen: Blinkt blau = bereit, Rot = Fehler
  2. WLAN-Verbindung: Lila LED bedeutet, der Writer sucht ein Netzwerk → mit «SpaceCaptain-NFC» verbinden und WLAN konfigurieren
  3. IP-Adresse in den Einstellungen korrekt?
  4. Neustart: Strom kurz trennen
